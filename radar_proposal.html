⬡ HEARTBEAT CYCLE FLOWCHART
Core Component Breakdown
PaperFetcher
Queries ArXiv API daily for topic-tagged papers. Queries Semantic Scholar filtering by competitor affiliation. Deduplicates, normalizes and caches metadata.
OpportunityMatcher
R2R Mode. LLM compares abstracts against repo_context files. Generates relevance score + reason. Creates GitHub issue when threshold exceeded.
ThreatDetector
SIGNAL Mode. Compares competitor papers against internal_briefs. Computes semantic overlap score. Fires Telegram alert when overlap > threshold.
OutputDispatcher
GitHub Issue Creator via REST API. Telegram Alerter via Bot API. WeeklyBriefComposer aggregates week's findings into a single digest message.
WeeklyBriefComposer
Aggregates all weekly findings. Formats structured Telegram digest including all opportunities, threats, and stats. Sends every Monday 7:30 AM.
Persistent Memory
Markdown/YAML files stored locally in ~/.openclaw/radar/. Three categories: repo_context/, internal_briefs/, competitor_profiles/.

SECTION 04 — ADVANCED CAPABILITIES
Section 04 ─────────────────────
Advanced Capabilities — Upgrades That Matter

Beyond the core pipeline, RADAR implements five differentiated capabilities that elevate it from a research scraper to a true AI research intelligence system. These upgrades are what make RADAR stand apart from any competing submission.

1
🕸️
Living Codebase Graph (vs. Static README)
RADAR reads the actual codebase — extracting architecture patterns, active TODOs, open PRs, and function signatures — then auto-generates repo context embeddings. The system knows your project better than the README does. Matching is against living code, not a stale summary.
AST parsing embeddings git diff ChromaDB
2
🔗
Citation Tracing — The "So What" Layer
When a paper matches a repo, RADAR traces which specific functions or modules in the codebase it's relevant to.

"This paper on quantization is relevant to model_compression.py → quantize_weights()"

This is 10× more actionable than "this paper is relevant to your project."
function-level matching LLM reasoning
3
⚖️
Patent Proximity Scoring for Threat Detector
After detecting a semantic threat, RADAR queries Google Patents / Lens.org to check whether the overlapping concepts are already filed as patents. This directly addresses the IP protection angle — and no other team will think of this.
Google Patents API Lens.org patent scoring
4
📈
Research Momentum Signals
Instead of per-paper alerting, a weekly trend layer detects when a cluster of papers from multiple institutions converges on a direction your team is working in. Strategic-level intelligence:
"3 papers from Google Brain, MIT & Stanford in 10 days all tackle on-device multimodal compression."
clustering topic modeling trend analysis
5
🔄
The Feedback Loop — System That Learns Over Time
RADAR currently fires and forgets. The Feedback Loop adds a reaction mechanism: when an engineer opens a RADAR GitHub issue and links it to a PR, RADAR learns this type of match was useful. When they close it as "not relevant," it learns the opposite.

After 30 days, the system has calibrated relevance scoring tuned to your specific team. No other hackathon project has this self-improvement loop.
GitHub webhook reinforcement signal relevance calibration 30-day learning window
